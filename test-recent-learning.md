# 最近学习功能测试指南

## 功能说明

最近学习组件现在已经连接到真实的 localStorage 数据，能够：

1. 显示用户最近学习的课程
2. 展示学习进度和当前学习位置
3. 提供快速继续学习的入口
4. 实时更新学习状态

## 测试步骤

### 1. 初始状态测试

- 打开应用，访问 `/study` 页面
- 如果没有学习记录，应显示"暂无学习记录"的空状态
- 点击"去发现课程"按钮应跳转到推荐页面

### 2. 开始学习测试

- 访问 `/step/python/python-basics` 页面
- 点击"开始学习"按钮
- 系统应自动注册课程并跳转到第一个步骤
- 返回 `/study` 页面，应能看到 Python 课程出现在最近学习列表中

### 3. 学习进度测试

- 在学习页面完成几个步骤
- 返回最近学习列表，应能看到进度更新
- "继续学习"按钮应能直接跳转到当前学习位置

### 4. 多课程测试

- 开始学习多个课程（如 JavaScript、HTML & CSS）
- 最近学习列表应按最后学习时间排序
- 最多显示5个最近课程

### 5. 数据持久化测试

- 刷新页面，学习记录应该保持
- 关闭浏览器重新打开，数据应该还在

## 预期行为

### 最近学习组件样式

- 美观的卡片式布局
- 显示课程图标、名称、语言标签
- 进度条和百分比显示
- 当前学习位置（第X章·步骤Y）
- 最后学习时间（刚刚、X分钟前、X小时前、昨天、X天前）
- 悬停效果和点击交互

### 数据更新

- 所有学习操作都会实时更新 localStorage
- 其他页面的最近学习组件会自动刷新
- 支持跨标签页的数据同步

## 调试信息

- 打开浏览器开发者工具
- 查看 Application > Local Storage > learning_status
- 可以看到详细的学习状态数据

## 可能的问题

1. 如果图标不显示，检查图片路径是否正确
2. 如果数据不更新，检查 localStorage 是否被禁用
3. 如果样式异常，检查 CSS 加载是否正确
